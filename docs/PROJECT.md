# meteor-city (운석 도시 — 3D 도시 파괴 샌드박스)

> 이 문서는 developer 가 이것만 보고 씬 구조·물리 스텝·도시 생성·운석/파괴 모듈·UI 를 짤 수 있게 쓴 **기술 스펙**이다.
> 최종 아트 방향(색·머티리얼·톤·구체 수치)은 designer 가 `DESIGN.md` 에서 확정한다. 여기선 "무엇이 고퀄로 보이게 하는가"의 요구와 구현 스코프까지만 못박는다.

---

## 한 줄 컨셉
매번 랜덤으로 지어지는 미니어처 3D 도시에 운석을 골라 떨어뜨리고, 건물이 물리로 부서지는 충격 순간을 슬로모로 감상·캡처하는 파괴 샌드박스 — **임팩트 순간의 스크린샷/GIF 와 "이 도시 부숴봐" 시드 링크**가 같이 퍼진다.

## 왜 이거 (탈출 근거)
- 버린 뻔한 방향: ① "물리 데모"(상자 몇 개 무너뜨리기 — 흔하고 공유 이유 없음) ② 포토리얼 도시 파괴(브라우저에서 못 만들거나 만들어도 렉으로 자폭) ③ 탑다운 2D 폭탄 게임(임팩트의 3D 쾌감이 없음)
- 이 컨셉의 뾰족한 각도: **"틸트시프트 미니어처 도시를 손가락으로 톡 건드려 부순다"**. 포토리얼을 포기하고 스타일라이즈드 PBR + 슬로모 + 카메라 셰이크 + 크레이터로 **"짓이기는 손맛"**에 전부를 건다. 매번 도시가 새로 생성돼 두 번 같은 파괴가 없다.
- 공유 유발 포인트: 사용자가 **임팩트 프레임(섬광·충격파 링·흩날리는 파편·크레이터)을 스크린샷/화면녹화**로 남기고, **`?seed=` 링크로 "이 도시 부숴봐"** 를 던진다. 받은 사람은 똑같은 도시를 열어 다른 각도/다른 운석으로 부수고 되던진다.

## 핵심 루프
1. 접속 → WASM/에셋 로드 후 **랜덤 미니어처 도시가 3/4 부감 각도로 이미 지어져 있음** (설명 없이 뭐 하는 앱인지 즉시 이해)
2. 하단 운석 팔레트에서 **운석 종류 선택** + 크기 토글
3. 커서를 도시 위로 → **조준 레티클 + 예상 크레이터 원 + 낙하 궤적 프리뷰**가 지면에 표시
4. **클릭 → 운석 낙하 → 충돌 → 파괴 시퀀스**(섬광·충격파·파편 강체·먼지·크레이터·카메라 셰이크)
5. **슬로모 토글**로 임팩트 순간을 늘려 감상·캡처 → 다시 떨어뜨리기 / **도시 재생성**(새 시드) / **시드 링크 공유**

## 기술 스택 결정 (근거 포함)

### 확정: Next.js(App Router+TS) + React-Three-Fiber(R3F) + drei + Rapier(WASM) + three-pinata. 무상태 100% 클라이언트.

| 항목 | 결정 | 근거 |
|------|------|------|
| 렌더러 | **three.js via R3F(`@react-three/fiber`) + `@react-three/drei`** | R3F 의 React 리컨실러는 애니메이션 루프 밖에서 돌아 오버헤드가 무시할 수준이고, drei 가 `OrbitControls`·`Environment(HDRI)`·`useGLTF`·`Instances`·성능 헬퍼를 바로 주어 생산성이 크게 앞선다. **단, 파편 강체는 React 로 마운트하지 않는다(아래 아키텍처 §"임퍼러티브 경계" 참고)** — 이 경계만 지키면 R3F 로도 헤비 물리가 안전하다. |
| 물리 엔진 | **Rapier3D (`@dimforge/rapier3d-compat`, WASM)** | 2026 기준 브라우저에서 가장 빠른 물리 엔진. SIMD 패키지는 2024 대비 2~5배 빠르고, 동적 BVH broadphase 로 수천 개 강체를 감당한다. convex hull 콜라이더·CCD(연속충돌)·sleep 을 기본 제공 → **파편 강체 대량 + 고속 운석**이라는 우리 요구에 정확히 맞는다. |
| (기각) cannon-es | 사용 안 함 | 순수 JS 라 셋업은 간단하나 대량 강체에서 Rapier 보다 느리고 CCD/sleep 안정성이 떨어진다. 우리 앱의 병목은 "파편 개수 × 물리 스텝"이라 성능이 곧 컨셉의 생사 → Rapier 로 확정. |
| React 래퍼 | **`@react-three/rapier` 는 씬 스캐폴드용으로만 제한적 사용** | 정적 콜라이더(지면·온전한 건물)까지는 편하지만, 수백 개 파편을 `<RigidBody>` 컴포넌트로 매 프레임 마운트/언마운트하면 리컨실러가 죽는다. **파편은 `useRapier()` 로 raw world 를 잡아 임퍼러티브로 생성**하고 `InstancedMesh` 로 그린다. (래퍼를 아예 안 쓰고 world 를 직접 만들어도 무방 — developer 재량. 단 파편은 무조건 임퍼러티브.) |
| 파괴/분할 | **three-pinata (MIT, TS, three≥0.158)** | 런타임 Voronoi 프랙처(3D/2.5D) + **impact-based fracture**(충돌점 주변으로 파편 집중) + 리프랙처를 제공. 권장 파편수 10~50개/회, manifold(watertight) 메시 필요 — 우리 건물은 박스형이라 조건 충족. 2.5D 모드가 3D 보다 빠름. 자체 프랙처를 짜지 않고 검증된 라이브러리 사용으로 스코프 절약. |
| 포스트프로세싱 | **`@react-three/postprocessing`(pmndrs) EffectComposer** | Bloom·DepthOfField(틸트시프트)·Vignette·ChromaticAberration 을 선언적으로. 임팩트 섬광의 bloom 스파이크가 "고퀄"의 절반. |
| 배포 | **Vercel 정적/Next 클라이언트** | 무상태 → `scripts/deploy.sh meteor-city` 그대로. |

> 로딩: rapier WASM + HDRI + three-pinata 는 async 초기화라 **로딩 상태 화면 필수**(§화면·상태).

### 임퍼러티브 경계 (developer 가 반드시 지킬 원칙)
- **React 리컨실러가 관리하는 것(정적/저빈도)**: Canvas, 라이팅, Environment, 카메라, OrbitControls, PostFX, **온전한 건물·지면·나무(InstancedMesh, 씬당 1회 빌드)**, HUD, AimReticle.
- **매 프레임 임퍼러티브로 도는 것(고빈도/대량)**: 운석 강체, **파편 강체 + 파편 InstancedMesh 싱크**, 파티클, 물리 스텝. 이건 `useFrame` 안에서 raw three/rapier 를 직접 만지고 React state 를 건드리지 않는다(리렌더 유발 금지).
- 물리 스텝은 **고정 timestep 1/60s 누산기(accumulator)** 로 렌더 fps 와 분리. 슬로모는 누산기에 곱하는 `timeScale`(예: 0.15)로 구현.

### 씬 그래프
- `sun`: 1× DirectionalLight(그림자 캐스터), 도시 bbox 에 딱 맞춘 좁은 shadow frustum, PCFSoftShadowMap 2048.
- `env`: drei `<Environment>` HDRI(부드러운 하늘/스튜디오) → PBR 반사·앰비언트. `toneMapping = ACESFilmic`.
- `ground`: 큰 평면(도로/잔디 텍스처 또는 버텍스 컬러) + Rapier 고정 cuboid 콜라이더.
- `cityStatic`: 머티리얼 패밀리별로 머지된 건물 지오메트리(few draw calls) + 나무 InstancedMesh. 각 온전한 건물은 Rapier **고정** cuboid 콜라이더 1개.
- `debris`: 파편 InstancedMesh(들), 아키타입/머티리얼별 1개씩. 강체는 임퍼러티브 풀.
- `fxLayer`: 파티클·충격파 링·데칼·섬광 스프라이트(풀링).

## 1. 절차적 도시 생성 스펙

**결정론 필수**: 모든 랜덤은 `mulberry32(seed)` 단일 스트림에서 **고정된 순서**로 뽑는다 → 같은 seed = 완전히 같은 도시. seed 는 `?seed=` 로 공유. `generateCity(seed)` 는 three 에 의존하지 않는 순수 함수(`CityModel` 데이터만 반환), `buildCityMeshes(model)` 가 지오메트리·콜라이더로 변환(테스트·재현 용이).

### 좌표/단위
- world unit ≈ 1m 감각. 셀 크기 `CELL = 20`, 도로 폭 `ROAD = 6`, 지면은 도시 bbox +여백.

### 생성 파이프라인 (고정 순서)
1. **그리드 결정**: `gridN = 6..10`(seed) 정사각 블록 격자. 도시 총 크기 ≈ `gridN*(CELL+ROAD)`.
2. **도로망**: 격자선마다 아스팔트 스트립(도로) 배치 + 중앙선/횡단보도는 텍스처. 확률적으로 일부 내부 도로를 스킵해 **블록 병합**(가변 블록 크기)으로 단조로움 제거.
3. **강(river)**: 확률 `~0.5` 로 도시를 가로지르는 **직선 또는 L 자 채널** 1개. 채널이 지나는 셀은 water 로 치환(반사 평면), 도로가 강을 만나면 **다리(bridge)** 로. (실물 유체 아님 — §스코프 컷)
4. **블록 타입 배정**: 각 (강 아닌) 셀에 가중 팔레트로 타입 지정 — `downtown`(고층 타워 밀집) / `residential`(저층 주택) / `commercial`(중층 상가) / `park`(녹지·연못·나무) / `plaza`(광장·낮은 구조물). 중심부일수록 downtown 가중↑(중심 거리 기반).
5. **로트 분할 & 건물 배치**: 각 건물 셀을 1~4 로트로 분할, 로트마다 마진 두고 건물 1개:
   - 아키타입(가중): `boxTower`(직육면체 고층) / `setbackTower`(층단 후퇴 고층) / `slab`(판상 중층) / `gableHouse`(박공 저층) / `warehouse`(넓고 낮음).
   - 크기: 발자국 `w,d` = 로트에 맞춰, 높이 `h` 는 블록 타입별 분포에서 — downtown `30~80`, commercial `12~28`, residential `4~12`, park 구조물 `3~6`.
   - 파사드: 별도 지오메트리 없이 **창문 emissive 그리드 텍스처 + 머티리얼 패밀리(유리타워/콘크리트/벽돌/도색)** 로 저비용 디테일.
6. **나무/프롭**: park·일부 도로변에 저폴리 나무(원통 줄기 + 콘/구 잎)를 **전부 하나의 InstancedMesh** 로. 가로등·차량 등은 **선택(스코프 컷 후보)**.

### 노출 파라미터 (MVP)
- `seed`(URL) 만 실질 공유값. 나머지(gridN·밀도·높이스케일·강 유무)는 seed 에서 파생 → 재현 보장.
- **선택 노출**: "밀도 프리셋"(듬성/보통/빽빽) 정도만 UI 노출 후보. 연속 슬라이더는 MVP 제외(공유 재현 복잡도↑).
- **재생성 버튼** = 새 랜덤 seed 뽑아 도시 재빌드 + URL 갱신. **리셋 버튼** = 같은 seed 로 파편만 제거하고 도시 원복.

## 2. 운석 시스템

### 프리셋 (`meteorPresets.ts`, MVP 4종 + 1 스트레치)
각 프리셋 필드: `mesh`(저폴리 icosphere + 노이즈 변위 or 간단 오서링), `radius`, `mass`(밀도×부피), `impactSpeed`, 블래스트 반경 배율 `R1/R2/R3`, 섬광/파티클 강도, 크레이터 크기, 머티리얼.

| 프리셋 | 성격 | 반경 | 질량/밀도 | 특징 |
|--------|------|------|-----------|------|
| `rocky` (암석형, 기본) | 균형 | 중 | 중 | 울퉁불퉁 회암 질감. 표준 크레이터·표준 블래스트 |
| `iron` (철운석) | 관통형 | 소 | 매우 높음 | 금속 반사 + 가열 emissive. 반경 작지만 **깊고 강한** 크레이터, 파편 임펄스↑ |
| `jagged` (각진 파쇄형) | 지저분 | 중 | 중 | 불규칙 각. 블래스트가 비대칭, 파편 방향 산발 |
| `comet` (혜성) | 광역 | 대 | 낮음(빙질) | 밝은 꼬리 + 다량 먼지. **넓고 얕은** 광역 블래스트, 섬광 최대 |
| `cluster` (군집) — **스트레치, MVP 제외** | 다중 충돌 | — | — | 낙하 중 분열해 여러 소충돌. 멀티 운석이라 무거움 → post-MVP |

### 파라미터 (MVP 최소화)
- **크기 토글 3단**(S/M/L): `radius`·`mass`·블래스트를 배율로. 연속 슬라이더 대신 3단(공유·튜닝 단순).
- 낙하각/속도는 **프리셋+크기에서 파생**, 기본 급강하(가파른 각). **커스텀 각도는 MVP 제외 스트레치.**

### 낙하 인터랙션
- **조준**: 카메라→커서 레이캐스트를 지면 평면과 교차 → 교차점에 `AimReticle`(지면 링 데칼 + 예상 크레이터 원 + 얇은 수직 가이드라인). 상공에 고스트 운석이 떠 있고 낙하 지점까지 **점선 궤적** 프리뷰.
- **드롭**: 클릭 시 타깃 상공(낙하각 오프셋)에서 운석을 **동적 강체**로 스폰, 초기 하향 속도 부여 → Rapier 중력으로 낙하 → 충돌 콜백에서 임팩트 시퀀스 트리거.
- **CCD 필수**: 운석은 고속이라 CCD(연속충돌) 켜지 않으면 지면/건물을 뚫는다(tunneling). developer 주의.
- 클릭 vs 드래그 구분(§조작)로 orbit 과 충돌 방지.

## 3. 파괴 모델 (핵심 고퀄 포인트)

포토리얼 시뮬레이션 대신 **"거리 기반 3-tier + 런타임 impact fracture + 전역 상한"** 으로 브라우저에서 확실히 구현 가능한 선을 잡는다. 임팩트 지점 거리 `d` 기준:

| 존 | 범위 | 처리 | 비용 |
|----|------|------|------|
| **Core** | `d < R1` | 건물을 **three-pinata impact-fracture** 로 10~30 조각으로 분할 → 각 조각을 Rapier convex-hull **동적 강체**로 스폰, 임팩트점 기준 방사 임펄스. **바닥에 크레이터 데칼**. | 높음 → **1충돌당 최대 프랙처 건물 수 상한**(예: 8동). 초과분은 Blast 처리로 강등 |
| **Blast** | `R1 ≤ d < R2` | 프랙처 안 함. 온전한 건물을 **고정→동적 전환** 후 방사 임펄스로 **넘어뜨리기/밀치기**(강체 1개라 저비용) + 몇 개 작은 청크만 튀김 | 중 |
| **Fringe** | `R2 ≤ d < R3` | 지오메트리 불변. **그을음 데칼 + 먼지** + 충격파 링 통과 시 창문 emissive 플래시 | 낮음 |

- `R1<R2<R3` 는 운석 프리셋·크기 배율로 스케일.
- **파편 강체**: 콜라이더는 trimesh 금지, **convex hull**(빠름). sleep 켜서 정지하면 시뮬 제외.
- **전역 파편 상한**(품질 티어별, 예 데스크탑 300 / 저사양 120): 초과 시 **가장 오래된(sleep) 파편부터 페이드아웃 후 despawn** → 반복 투하해도 씬이 깨끗하게 유지되고 fps 방어. **영구 잔해 유지는 안 함**(§스코프 컷).
- **리프랙처**: 기본 1레벨(파편을 또 부수지 않음). three-pinata 가 지원하나 MVP 는 단일 레벨.
- **프랙처 히치 방어**: 런타임 프랙처는 순간 부하 → 1충돌당 프랙처 건물 수 상한 + 가능하면 three-pinata 의 `onFragment` 콜백으로 조각 스폰을 프레임에 분산. 2.5D 모드가 빠르나 건물은 3D voronoi 가 보기 좋음 → **기본 3D, 저사양 티어는 2.5D 로 폴백**.

## 4. 이펙트 / "고퀄" 정의 (요구 수준; 수치는 designer 확정)

"고퀄로 보이게 하는" 요소를 우선순위로:
1. **임팩트 섬광 + Bloom 스파이크**: 충돌 순간 밝은 emissive 구 + PostFX Bloom 순간 증폭 → 화면이 번쩍. (스크린샷 임팩트의 핵심)
2. **충격파 링**: 지면을 따라 퍼지는 확장 링(반경 스케일 + 알파 폴오프 셰이더). 통과하는 건물 창문 emissive 순간 점멸.
3. **먼지/연기 파티클**: 임팩트점에서 확장·상승·페이드하는 빌보드 파티클(InstancedMesh/Points 풀). 운석별 양·색 조절(혜성 최대).
4. **파편 발광 냉각**: 파편 emissive 를 뜨겁게 시작해 점점 어둡게(코어존 한정).
5. **크레이터 + 그을음 데칼**: 지면 투영 원형 데칼(크레이터 텍스처 + 스코치). **실제 지형 함몰 지오메트리는 MVP 제외**(데칼로 대체, 아주 약한 버텍스 dent 는 스트레치).
6. **카메라 셰이크**: 트라우마 기반 감쇠 셰이크(거리·운석 크기 비례).
7. **틸트시프트 DOF + Vignette + 약한 ChromaticAberration**: 미니어처 장난감 도시 룩을 만들고 임팩트 시 CA/비네트 펄스. **틸트시프트를 이 앱의 시그니처 룩 후보로 designer 에게 제안** — "장난감 도시를 부순다"는 컨셉과 정확히 맞고 캡처가 예뻐진다.

**스타일 방향(planner 판단, designer 최종)**: **포토리얼 기각 → 스타일라이즈드 PBR(클린 지오메트리 + 좋은 라이팅/머티리얼 + 강한 PostFX)**. 텍스처를 무겁게 깔지 않고 라이팅·틸트시프트·물리 손맛으로 "고퀄"을 낸다. 포토리얼은 브라우저에서 자폭 위험 + AI 티 나는 제네릭으로 흐르기 쉬움.

**사운드**: 임팩트 SFX(저음 쿵 + 유리/콘크리트 파쇄 + 잔해)는 **화면녹화 공유력을 크게 올림 → 강력 권장이나 MVP 필수 아님**(nice-to-have, 음소거 기본 + 토글).

## 7. 상태 저장

- [x] **무상태 (Vercel+git 로 끝)** / [ ] backend 필요
- 근거: 도시 생성·물리·파괴·이펙트가 전부 클라이언트(three/rapier/WASM). 공유·재현은 **`?seed=`(+선택 `?type=`) URL 쿼리**로 완결 — 서버·DB·저장 로직 불필요. 개인 마지막 설정(운석 종류·크기·품질)은 **localStorage** 로 충분. 사용자 간 공유 상태/랭킹/갤러리가 없으므로 backend 조건에 해당 없음 → Vercel 정적 배포만으로 끝.
- URL 스킴: 공유 링크는 우리 도메인 쿼리만 사용(외부 입력을 `dangerouslySetInnerHTML` 등에 넣지 않음). seed 는 정수 파싱·범위 검증, type 은 화이트리스트 검증.

## 8. MVP 범위

### 포함
- 시드 결정론 절차적 도시(도로 격자·블록 타입·건물 아키타입·나무·강/다리) + 재생성/리셋
- 운석 4종(rocky/iron/jagged/comet) × 크기 3단, 조준 레티클 + 예상 크레이터 + 궤적 프리뷰, 클릭 드롭(CCD)
- 거리 3-tier 파괴(Core 런타임 프랙처 강체 / Blast 토플 / Fringe 데칼), 전역 파편 상한 + despawn
- 이펙트: 섬광+Bloom, 충격파 링, 먼지 파티클, 크레이터/그을음 데칼, 카메라 셰이크, 틸트시프트 DOF/Vignette/CA
- Orbit 카메라 + 슬로모, HUD(팔레트/크기/슬로모/재생성/리셋/공유), 품질 티어 자동 강등
- `?seed=`(+`?type=`) 공유, localStorage 설정 기억, 로딩/에러 화면

### 명시적 제외 (v1 안 함 — "다 넣다 못 만드는" 방지)
- **멀티 운석 동시/운석 샤워(cluster)** — 다중 충돌은 물리 폭발적, post-MVP
- **실물 유체 시뮬레이션** — 강/연못은 정적 반사 평면, 운석 낙하 시 스플래시 파티클만(진짜 유체 X)
- **영구 잔해 유지** — 파편은 상한/시간으로 despawn, 무한 누적 X
- **실제 지형 함몰 크레이터 지오메트리** — 데칼로 대체(약한 버텍스 dent 는 스트레치)
- **다단 리프랙처(파편을 또 부수기)** — 단일 프랙처 레벨
- **커스텀 낙하각/연속 속도 슬라이더** — 프리셋+크기 3단 파생
- **사운드 필수화** — nice-to-have(음소거 기본 토글)
- **리플레이/카메라 자동 시네마틱 저장·영상 내보내기** — 스트레치
- **계정/저장/갤러리/랭킹/좋아요** — backend 필요, 아예 안 함
- **포토리얼 텍스처·베이크드 GI·차량/보행자·주야 사이클** — 스타일라이즈드로 대체
- **모바일 최적화** — 동작만, 자동 저사양 강등

## 9. 화면 · 상태

- **로딩**: rapier WASM + HDRI + 도시 생성 중 → 테마 있는 로더(예: 낙하하는 운석/카운트다운). 진행 표시.
- **Ready(대기)**: 도시 빌드 완료, HUD 노출, 커서 따라 조준 레티클. 빈 화면 없음(항상 도시 존재).
- **Aiming**: 레티클 + 예상 크레이터 원 + 궤적 프리뷰 표시.
- **Impact(액티브 시뮬)**: 낙하→충돌→이펙트 재생→파편 정착. 슬로모 홀드로 늘려 감상.
- **Post-impact(대기)**: 파편/크레이터 남은 채 재투하 가능. 상한 초과 시 오래된 파편 페이드아웃.
- **에러**: WebGL/WASM 미지원·로드 실패 → 친절한 폴백("이 브라우저에선 3D 파괴를 못 돌려요") + 정적 이미지. 공유 URL seed 파싱 실패 → 기본 랜덤 도시로 폴백 + 안내.
- **공유 링크 진입**: `?seed=` 디코딩 → 동일 도시 재현 + (있으면) `?type=` 운석 선택 상태로 시작.
